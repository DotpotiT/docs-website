Introduction

Welcome to the website documentation for Dotpot IT! Our website provides a range of features, including a login and registration system, a job portal, a "Recent Works" showcase, a blog section, a customized services ordering system, a chat app for communicating with customer service representatives, and themes for users to customize the website's appearance.

This documentation provides a comprehensive guide to the structure, functionality, and features of our website. It is designed to help developers understand how the website works and how to manage it effectively.

In this documentation, you will find information about the website's design, layout, content, functionality, security, and performance. You will also find detailed instructions on how to use the website's features, such as registration, login, search, and checkout.

To help you navigate this documentation, we have organized it into different sections, each covering a specific aspect of the website. You can use the table of contents or search function to find the information you need quickly and easily.

We have also included code samples, technical specifications, and API documentation for developers who want to integrate the website with third-party tools and services.

If you have any questions or encounter any issues while using our website, please refer to our FAQs and troubleshooting guides. If you still need assistance, you can contact our support team for further help.

We are committed to providing high-quality and up-to-date documentation, so please check back regularly for updates and improvements.

Thank you for using Dotpot IT, and we hope you find this documentation helpful!

System Requirements

The requirements for a computer to develop and run our website are as follows:

  • Operating System: Windows, macOS, or Linux
  • Processor: 2 GHz or faster multi-core processor
  • RAM: 8 GB or more
  • Hard Disk: At least 20 GB of free space
  • Graphics card: Any modern graphics card with support for OpenGL
  • Web server software: Apache, Nginx or Microsoft IIS
  • Database server software: MongoDB
  • Code Editor: A code editor such as Visual Studio Code, Sublime Text, or Atom

Installation

To install and set up our website, you will need to follow these general steps:

Clone Website

  • Clone the repository, run the following command: git clone github.com:DotpotiT/website.git

API Setup

  • Start the client side by running the following command: cd api
  • To install the dependencies, run the following command: npm install
  • Start the server by running the following command: npm start

Client Setup

  • Start the client side by running the following command: cd client
  • To install the dependencies, run the following command: npm install
  • If you run into errors, run the following command: npm install --legacy-peer-deps
  • Start the server by running the following command: npm run dev
  • Open the project on browser at [Server] http://localhost:port_number

Admin Setup

  • Start the client side by running the following command: cd admin
  • To install the dependencies, run the following command: npm install
  • Start the server by running the following command: npm run dev
  • Open the project on browser at [Server] http://localhost:port_number

APIs

We used RESTful API in our website. RESTful APIs offer several benefits over other types of APIs, including scalability, flexibility, security, and compatibility. They are designed to handle a large number of requests without any impact on performance and can support a wide range of data formats. RESTful APIs can also be secured using a variety of authentication mechanisms, ensuring that data is protected in transit.We created the APIs using NodeJs, ExpressJs, Axios.

Google Signup Endpoint


This endpoint is used to sign up a new user to your application using their Google account credentials.

Request


URL: api/auth/google

HTTP Method: POST

Headers: Content-Type: application/json

Body:

Field Type Required Description
code string Yes The authorization code obtained from Google after the user grants permission for your app to access their Google account.

Example request body:

{
"code": "google auth code"
}

Response


Success Status Code: 200 OK

Content Type: application/json

Response Parameters:

Field Type Description
success boolean Indicates whether the operation was successful or not.
user object An object containing information about the newly created user.
token string A JWT token that can be used to authenticate subsequent requests.


Example Response:

{
"success": true,
"user": {
"_id": "user id",
"username": "user name",
"email": "user email",
"photo": "user photo URL",
},
"token": "JWT token", }

Error Responses

Status Code: 500 Internal Server Error

Content Type: application/json

Description: Returned if there is an error during the authentication or database operation.

Body:

Field Type Description
error string A message describing the error that occurred.

Example response:

{
"error": "error message"
}

Status Code: 401 Unauthorized

Content Type: application/json

Description: Returned if the user email is not verified.

Body

Field Type Description
error string A message indicating that the user is not authorized.

Example response:

{
"error": "Unauthorized"
}


Reset Password


Initiates a password reset request for a user.

URL: /api/auth/reset-password

HTTP Method: POST

Authentication: No authentication required

Request Parameters:

Parameter Type Description
email string The email address of the user to reset

Response Parameters:

Parameter Type Description
success boolean Whether the request was successful
message string A message describing the result

Response Codes:

HTTP Code Description
200 Successful
404 User not found
500 Server error

Example Request:

{
"email": "johndoe@example.com",
}


Example Response:

{
"success": true,
"message": "Email sent, please check your mail!"
}

Verify Reset Password

Verifies a user's reset password token and updates their password in the database.

Request

Method: POST

Headers:

Content-Type: application/json

Path Parameters:

resetToken (string): The reset password token received by the user.

Request Body:

{
"password": "string"
}

password (string): The new password to set.

Responses

Success (200 OK)

{
"success": true,
"message": "Password reset successfully"
}

Error (404 Not Found)

{
"success": false,
"message": "Invalid or expired reset token"
}

Error (500 Internal Server Error)

{
"success": false,
"message": "Something went wrong"
}

Example

Request

POST api/auth/reset-password/6eb990b423b63dc334623a8a8b1b4eef7bfabbfd
Content-Type: application/json
{
"password": "newPassword123"
}

Response:

HTTP/1.1 200 OK
Content-Type: application/json
{
"success": true,
"message": "Password reset successfully"
}

AWS Deployment

  • Create an AWS account if you don't have one already.
  • Launch an EC2 instance with the desired specifications, such as the operating system, storage, and network configuration. You can use an Amazon Machine Image (AMI) with pre-installed software, or you can manually install the required software on the instance.
  • Set up security groups to manage inbound and outbound traffic to the instance. Ensure that the necessary ports are open for your application to function properly.
  • Connect to the instance using SSH or a remote desktop client, depending on your instance's operating system.
  • Install PM2 on the instance by running the following command: npm install -g pm2
  • Clone the DotpotiT website repository from GitHub onto the instance. git clone https://github.com/DotpotiT/dotpotit.git
  • Go to API and install the necessary dependencies cd api npm install
  • Start the API server using PM2. pm2 start npm --name "api" -- start
  • Serve the client-side application using the nginx.
  • Install the necessary dependencies, such as Node.js and other packages required by the application for client. cd client npm install --legacy-peer-deps
  • Build the client-side application using the following command: npm run build
  • Serve the Client dist folder using PM2. pm2 start --name "client" "npx serve dist -s -l 5173"
  • Set up a reverse proxy to route traffic from the web server to the API server running on port 8800.
  • Test the application to ensure it's functioning as expected.
  • Create a domain name for the application using Amazon Route 53.
  • Configure the DNS records to point to the instance's public IP address.
  • Set up SSL/TLS certificates using Amazon Certificate Manager to encrypt traffic to and from the application.
  • Finally, monitor the application's performance and security, and make any necessary updates or changes.
