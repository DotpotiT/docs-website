Installation

To install and set up our website, you will need to follow these general steps:

Clone Website

  • Clone the repository, run the following command: git clone github.com:DotpotiT/website.git

API Setup

  • Start the client side by running the following command: cd api
  • To install the dependencies, run the following command: npm install
  • Start the server by running the following command: npm start

Client Setup

  • Start the client side by running the following command: cd client
  • To install the dependencies, run the following command: npm install
  • If you run into errors, run the following command: npm install --legacy-peer-deps
  • Start the server by running the following command: npm run dev
  • Open the project on browser at [Server] http://localhost:port_number

Admin Setup

  • Start the client side by running the following command: cd admin
  • To install the dependencies, run the following command: npm install
  • Start the server by running the following command: npm run dev
  • Open the project on browser at [Server] http://localhost:port_number

APIs

We used RESTful API in our website. RESTful APIs offer several benefits over other types of APIs, including scalability, flexibility, security, and compatibility. They are designed to handle a large number of requests without any impact on performance and can support a wide range of data formats. RESTful APIs can also be secured using a variety of authentication mechanisms, ensuring that data is protected in transit.We created the APIs using NodeJs, ExpressJs, Axios.

Auth Router


Register User


Registers a new user.

URL: /api/register

HTTP Method: POST

Authentication: No authentication required

Request Parameters:

Parameter Type Description
username string The user's username
email string The user's email address
password string The user's password

Response Parameters:

Parameter Type Description
token string A JSON Web Token (JWT) that can be used for authentication
user object The registered user's details, without the password field

Response Codes:

HTTP Code Description
200 User successfully registered
400 User already exists or invalid input
500 Server error

Example Request:

{
"username": "johndoe",
"email": "johndoe@example.com",
"password": "password123"
}


Example Response:

{
"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..",
"user": {
"id": "616cc7fbf7669b05c87eb4d5",
"username": "johndoe",
"email": "johndoe@example.com"
}
}

Please note that the actual implementation of this endpoint may vary depending on your specific use case and requirements. This is just an example to help you get started with documenting your API.

NodeJs

Node.js is known for its ability to handle large volumes of requests and data-intensive applications, making it a popular choice for building scalable and high-performance websites. We have implemented nodeJs to use javascript on both the server-side and client-side of the website to improve the website's responsiveness and overall user experience.

Intergrations




Figure-1


Integrating ReactJS, Node.js, MongoDB, and Socket.IO on our website can provide a powerful and dynamic user experience. ReactJS is a popular JavaScript library for building user interfaces, Node.js is a JavaScript runtime that allows for server-side scripting and building scalable network applications, MongoDB is a NoSQL database that allows for scalable and flexible data storage, and Socket.IO enables real-time communication between the server and client (Figure-1). By integrating these technologies, you can build a responsive and interactive website that can update data in real-time, handle large amounts of data, and provide a seamless user experience. For example, you can use Socket.IO to send and receive messages between users in real-time, while MongoDB can handle the storage and retrieval of this data. Meanwhile, Node.js can provide a powerful backend server for handling requests and managing data, while ReactJS can provide a smooth and engaging user interface that updates dynamically as data changes.

Utilities

Here is a list of some common and popular utilities that we have used in our website:

npm

npm stands for "Node Package Manager." It is a package manager for the JavaScript programming language and is primarily used for managing dependencies in Node.js projects.

webpack

Webpack is a popular module bundler for JavaScript applications. It allows developers to bundle multiple modules and assets into a single file or set of files for deployment to a browser or server. Webpack offers many features, such as code splitting, hot module replacement, and tree shaking, which help optimize the performance and speed of web applications.

JWT

JWT or "JSON Web Token" is a popular open standard for creating and transmitting secure authentication tokens between parties. JWTs are used for securely transmitting information between parties, such as a client and server, without the need for session management on the server. This is done by digitally signing and encoding the information into a JSON object, which can then be sent as a compact string.

AWS Deployment

  • Create an AWS account if you don't have one already.
  • Launch an EC2 instance with the desired specifications, such as the operating system, storage, and network configuration. You can use an Amazon Machine Image (AMI) with pre-installed software, or you can manually install the required software on the instance.
  • Set up security groups to manage inbound and outbound traffic to the instance. Ensure that the necessary ports are open for your application to function properly.
  • Connect to the instance using SSH or a remote desktop client, depending on your instance's operating system.
  • Install PM2 on the instance by running the following command: npm install -g pm2
  • Clone the DotpotiT website repository from GitHub onto the instance. git clone https://github.com/DotpotiT/dotpotit.git
  • Go to API and install the necessary dependencies cd api npm install
  • Start the API server using PM2. pm2 start npm --name "api" -- start
  • Serve the client-side application using the nginx.
  • Install the necessary dependencies, such as Node.js and other packages required by the application for client. cd client npm install --legacy-peer-deps
  • Build the client-side application using the following command: npm run build
  • Serve the Client dist folder using PM2. pm2 start --name "client" "npx serve dist -s -l 5173"
  • Set up a reverse proxy to route traffic from the web server to the API server running on port 8800.
  • Test the application to ensure it's functioning as expected.
  • Create a domain name for the application using Amazon Route 53.
  • Configure the DNS records to point to the instance's public IP address.
  • Set up SSL/TLS certificates using Amazon Certificate Manager to encrypt traffic to and from the application.
  • Finally, monitor the application's performance and security, and make any necessary updates or changes.
